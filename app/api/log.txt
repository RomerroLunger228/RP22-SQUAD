1. ПОКУПКА ПОДПИСКИ

  Шаг 1: Просмотр доступных планов

  GET /api/subscription-plans
  Что делает:
  - Получает все планы подписок из БД, сгруппированные по тирам (Default/Premium)
  - Возвращает реальные названия услуг, цены, скидки, количество бесплатных визитов
  - Связывает планы с конкретными услугами через service_id

  Шаг 2: Создание Stripe Checkout

  POST /api/subscription/purchase
  Body: { planId: 30, userId: 1 }
  Что делает:
  - Проверяет, нет ли у пользователя активной подписки
  - Получает план подписки с полной информацией об услуге
  - Создает Stripe Checkout Session с реальным названием услуги
  - Возвращает URL для перехода к оплате

  Шаг 3: Webhook активации подписки

  POST /api/stripe/webhooks (автоматический)
  Что делает:
  - Получает событие checkout.session.completed от Stripe
  - Проверяет metadata.type === 'subscription'
  - Удаляет старые подписки пользователя
  - Создает новую активную подписку с датой истечения
  - Устанавливает free_visits_used = 0

  2. ПРОВЕРКА СТАТУСА ПОДПИСКИ

  Запрос статуса подписки

  GET /api/user/subscription?userId=1
  Что делает:
  - Ищет активную подписку пользователя
  - Проверяет срок действия (expires_at vs текущая дата)
  - Если истекла - меняет статус на 'expired'
  - Рассчитывает оставшиеся бесплатные визиты
  - Возвращает полную информацию о подписке

  Пример ответа:

  3. ПРОЦЕСС ЗАПИСИ С ПОДПИСКОЙ

  Шаг 1: Отображение льгот в UI

  Frontend компонент SubscriptionBenefits:
  - Загружает статус подписки пользователя
  - Показывает карточку с активной подпиской
  - Отображает количество бесплатных визитов и процент скидки

  Шаг 2: Выбор услуги

  В компоненте Services:
  - Пользователь видит доступные услуги
  - Если есть подписка - отображается информация о льготах
  - Подписка применяется только к совместимым услугам

  3️⃣ Расчет цены с учетом подписки

  Детальные проверки в calculateAppointmentPrice:

  a) Проверка активной подписки

  // getActiveUserSubscription()
  const subscription = await prisma.subscription.findUnique({
    where: { user_id: userId },
    include: { subscription_plans: { include: { services: true } } }
  });

  // Проверка срока действия
  const isExpired = subscription.expires_at < new Date();
  if (isExpired) {
    await prisma.subscription.update({
      where: { user_id: userId },
      data: { status: 'expired' }
    });
    return null; // Подписка неактивна
  }

  b) Проверка совместимости услуги

  // canUseSubscription()
  // Проверка 1: Прямое соответствие service_id
  if (plan.service_id === serviceId) {
    return true; // Точное совпадение
  }

  // Проверка 2: По типу услуги
  const service = await prisma.services.findUnique({ where: { id: serviceId } });

  let serviceType;
  if (service.name.includes('борода') || service.name.includes('коррекция')) {
    serviceType = 'haircut_beard';
  } else if (service.name.includes('стрижка')) {
    serviceType = 'haircut';
  }

  return plan.service_type === serviceType;

  c) Определение типа льготы

  const freeVisitsUsed = subscription.free_visits_used || 0;
  const freeVisitsAvailable = plan.free_visits_count || 0;

  // Приоритет 1: Бесплатные визиты
  if (freeVisitsUsed < freeVisitsAvailable) {
    return {
      type: 'free',
      price: 0,
      originalPrice: service.pl_price
    };
  }

  // Приоритет 2: Скидка
  const discountPercentage = plan.discount_percentage || 0;
  const discountAmount = Math.round(originalPrice * discountPercentage / 100);
  const discountedPrice = originalPrice - discountAmount;

  return {
    type: 'discount',
    price: discountedPrice,
    originalPrice,
    discountAmount
  };

  4️⃣ Создание записи

  5️⃣ Обновление счетчиков подписки

  4. ОТОБРАЖЕНИЕ В ПРОФИЛЕ

  Компонент SubscriptionCard

  Что показывает:
  - Тип подписки (Default/Premium)
  - Реальное название услуги из БД
  - Оставшиеся бесплатные визиты
  - Процент скидки
  - Дату истечения
  - Прогресс-бар использованных визитов

  ---
  ИТОГОВАЯ ЛОГИКА ПРИОРИТЕТОВ:

  1. Проверка активности подписки (срок действия)
  2. Проверка совместимости услуги (service_id или service_type)
  3. Применение бесплатных визитов (если есть неиспользованные)
  4. Применение скидки (если бесплатные визиты кончились)
  5. Обычная цена (если подписки нет или услуга не подходит)

  Вся система работает автоматически и прозрачно для пользователя! 🎉
